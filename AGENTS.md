# Agent notestructio

Write like a software engineer shipping real products, not like an academic paper. Prefer short sentences and concrete examples over theory dumps.

Do not use em dashes, do not write the two-character ASCII arrow that looks like leads-to in prose, and avoid cramming extra thoughts inside parentheses. If you need a second clause, use a comma, a new sentence, or a bullet list.

Match existing code style in each file. These rules apply to comments, docs, commit messages, and user-facing strings. If an assignment quotes exact required wording, keep that wording.

Stay clear and direct. Skip stiff formal openings and filler.
